# 隅灵 (Yú Líng) 桌宠分阶段开发计划

- **作者**: Codex Agent

## 阶段概览
| 阶段 | 目标 | 预计耗时 |
| --- | --- | --- |
| 阶段 0 · 环境基线 | 搭好跨平台开发/构建环境，建立仓库与基本 CI | 1-2 天 |
| 阶段 1 · 幻影显形 | 完成透明窗口、托盘、PlatformBridge 基础 | 3-4 天 |
| 阶段 2 · 灵动交互 | 搭建 Motion 系统、跨屏拖拽、交互菜单 | 4-5 天 |
| 阶段 3 · 神兽之语 | Rust 提醒内核 + 灵盒提醒 UI | 4-5 天 |
| 阶段 4 · 发布验收 | 跨平台打包、性能/稳定性验证、签名 | 3 天 |

每个阶段内的任务清单均采用 Todo 形式，执行时由开发者勾选。建议按阶段顺序推进，不跨阶段并行，除非另有资源。

## 阶段 1 · 幻影显形（壳体 + PlatformBridge 基础）
- [ ] `pnpm create tauri-app` 初始化项目，改为 Vue 3 + TypeScript + Vite。
- [ ] 在 `tauri.conf.json` 启用 `transparent=true`、`decorations=false`、`alwaysOnTop=true`，验证窗口透明度。
- [ ] Rust `platform_bridge` 模块：
  - [ ] 定义 `PlatformMonitors` 结构；调用 `available_monitors()` 并通过 `tauri::command` 暴露。
  - [ ] 监听显示器变更事件（macOS `NSApplicationDidChangeScreenParametersNotification` / Windows `DisplayChange`），向前端 `emit platform:monitors-updated`。
  - [ ] 实现 `logical_to_physical` / `physical_to_logical` 辅助函数。
- [ ] 托盘与开机自启：
  - [ ] 添加系统托盘菜单（显示主宠物窗口、退出）。
  - [ ] 提供 `set_autostart` 命令（macOS LaunchAgent + Windows 注册表）。
- [ ] 创建最小前端容器，去除 body 背景，显示占位元素。
- [ ] 写初步手动测试表（透明窗口、置顶、托盘交互）并完成验证。

**完成判定**：透明置顶窗口可在双平台启动，托盘可显示/退出，监视器列表在热插拔时自动更新。

## 阶段 2 · 灵动交互（动画 + 拖拽 + 菜单）
- [ ] 定义 `assets/motion/manifest.json` 模板与 `MotionManifest` TypeScript 类型。
- [ ] 引入 PixiJS（或选定渲染器），实现 `MotionController`：
  - [ ] 解析 Manifest、加载序列帧（先用 CSS 几何或简易 PNG）。
  - [ ] 以 `requestAnimationFrame` + 时间差驱动插值，并支持 60/12 fps 切换。
- [ ] 交互层：
  - [ ] 实现拖拽监听（鼠标/触控），拖拽中切换 `drag` 状态。
  - [ ] 与 PlatformBridge 提供的坐标转换协作，松手后执行 `spring easing` + 边缘吸附。
  - [ ] 单击反应动画、双击打开“灵盒”占位菜单。
- [ ] Pinia 状态：`currentState`、`powerMode`、`windowPosition`，含调试面板。
- [ ] 低功耗模式：
  - [ ] 切换时调整 MotionController 目标 fps、减少位置监听频率。
  - [ ] 监听 `platform:session` 事件，锁屏 5 分钟后切 `sleep` 状态。
- [ ] 灵盒 UI（占位版）：显示状态信息、手动切换低功耗模式。
- [ ] 更新测试清单：覆盖拖拽跨屏、吸附、模式切换；录制短视频留档。

**完成判定**：角色可丝滑显示 idle/react/drag，跨屏拖拽无跳跃，灵盒可打开并切换模式。
