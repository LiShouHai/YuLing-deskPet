# 隅灵 (Yú Líng) 沉浸式互动桌宠 - 设计文档

- **版本**: 1.0.0
- **日期**: 2026-04-01
- **作者**: Codex Agent

## 1. 背景与目标
在繁忙的桌面环境中提供一个透明、置顶、可拖拽的数字宠物，通过丝滑动画与轻量提醒陪伴用户。第一阶段聚焦最小可行特性：角色展示、动画状态切换、跨屏拖拽、手动提醒配置与触发，同时为后续扩展（AI 对话、外部服务同步）打下模块化基础。

## 2. 系统概览
```
+--------------------+        +------------------------+
|  Tauri Shell       |  IPC   |  Vue 3 Frontend        |
|  (Rust)            +<------>+  Motion + UI Layer     |
|  - PlatformBridge  |        |  - MotionController    |
|  - ReminderEngine  |        |  - InteractionLayer    |
+---------+----------+        +-----------+------------+
          |                                |
          | native APIs                    | canvas/webgl
          v                                v
   OS services (monitors, DPI,      Animated character,
   notifications, session)          menus, reminder UI
```

核心模块：
1. **Tauri Shell**：透明无边框窗口、托盘、开机自启、事件总线。
2. **PlatformBridge**：封装多屏、DPI、锁屏/唤醒、通知等系统能力，向前端广播统一事件。
3. **ReminderEngine**：基于 Rust + SQLite 的事件驱动提醒调度器，负责持久化、定时、通知触发。
4. **MotionController**：前端状态机，解析动画清单，驱动序列帧或骨骼动画。
5. **InteractionLayer**：处理点击/拖拽/双击，管理提醒列表菜单。

## 3. 模块设计
### 3.1 PlatformBridge
- **MonitorService**：调用 `window.available_monitors()` + 平台事件（Windows `DisplayChange`、macOS `NSApplicationDidChangeScreenParametersNotification`）生成 `PlatformMonitors` 数据，并提供 `map_point_to_screen`、`clamp_to_work_area`、`logical<->physical` 转换。
- **SessionService**：监听 `WM_POWERBROADCAST`/`WTSRegisterSessionNotification` 与 `NSWorkspace` 会话事件，统一发出 `platform:session`（`locked`/`unlocked`）。
- **NotificationService**：封装 Windows Toast API 与 macOS UserNotifications，接口 `show_notification(payload)`，失败时回退为前端气泡提示。
- **Tray & Autostart**：提供 `register_tray(menu)` 与 `set_autostart(enabled)`；内部分别写入 Windows Run 注册表与生成 LaunchAgent plist。

### 3.2 ReminderEngine
- **数据模型**：
```ts
interface Reminder {
  id: string;
  title: string;
  scheduled_at: i64; // UTC ms
  repeat_rule?: RepeatRule; // daily/weekly/custom
  snooze_minutes?: u32;
  status: 'pending' | 'done' | 'muted';
}
```
- **存储**：SQLite 数据库 `reminders.db`；使用 `serde` + `rusqlite` 序列化。
- **调度**：启动时加载所有提醒→建立 `DelayQueue`（`tokio::time::sleep_until`）。新增/编辑/删除时实时重排。收到 `platform:session` 的 `locked` 时暂停触发；`unlocked` 后计算漏掉的任务并立即触发补偿事件。
- **触发流程**：
  1. 调用 `NotificationService.show_notification`。
  2. `app_handle.emit("reminder:fired", payload)` 给前端。
  3. 如有重复规则则生成下次任务；否则将 `status` 置 `done`。

### 3.3 前端子系统
- **MotionManifest**：`assets/motion/manifest.json` 声明状态、fps、循环、可中断性；`MotionController` 读取后以时间差驱动 WebGL/Canvas 渲染。
- **AnimationAdapter**：统一封装序列帧（Spritesheet）与骨骼（Spine/GDScene）渲染，输出 `currentFrame`, `bbox`，供交互层命中测试。
- **InteractionLayer**：
  - 单击→`react` 状态；
  - 长按→`drag`，根据 `PlatformBridge.map_point_to_screen` 计算跨屏位置，并在松手后执行 `spring easing` 滑行 + 吸附（参考 `work_area`）。
  - 双击→弹出提醒列表菜单，允许创建/编辑提醒与低功耗开关。
- **低功耗策略**：Pinia 状态 `powerMode` 控制目标 fps（60→12），同时降低渲染 tick、位置轮询频率（实时→500ms），并在会话锁定或 5 分钟无交互时进入 `sleep` 状态，暂停动画。

## 4. 动画与资源
- 资源放置：`assets/motion/<state>/<frame>.png` + `metadata.json`。示例：`assets/motion/idle/idle_0001.png`。
- Manifest 示例：
```json
{
  "idle": {"fps": 24, "loop": true, "interruptible": true},
  "drag": {"fps": 30, "loop": true, "interruptible": false},
  "reminder_pulse": {
    "fps": 12,
    "loop": true,
    "overlay": "glow",
    "priority": 80
  }
}
```
- 状态优先级：`drag > reminder_pulse overlay > react > idle > sleep`。`reminder_pulse` 以叠加层实现，保证提醒时角色仍维持主体动作。
- 渲染策略：初期采用 PixiJS（WebGL）；时间驱动插值确保高/低帧率都平滑。对高分屏按 DPR 缩放绘制，避免跨屏模糊。

## 5. 提醒系统流程
1. 用户在提醒列表菜单创建提醒，前端校验字段后调用 `tauri.invoke("reminder_create", payload)`。
2. ReminderEngine 写入 SQLite 并重排 `DelayQueue`，通过事件 `reminder:updated` 返回最新任务列表。
3. 到点时触发系统通知 + 前端事件。前端弹出毛玻璃言弹气泡，按钮提供 `完成` / `稍后提醒`。
4. 用户选择动作→调用 `reminder_complete` 或 `reminder_snooze`，Engine 更新状态并广播。
5. 所有提醒状态同步到 Pinia，本地 UI 与托盘菜单保持一致。

## 6. 平台特性与安全
- **透明窗口**：`tauri.conf.json` 启用 `decorations=false`、`transparent=true`、`alwaysOnTop=true`，macOS 额外调用 `setIsOpaque(false)` + `setBackgroundColor(NSColor.clear)`；Windows 设置 Layered Window，实现点击穿透时使用命中测试回调。
- **权限**：`tauri.conf.json > security > allowlist > window` 开启 monitor API；macOS Info.plist 添加 `NSMicrophoneUsageDescription` 等未来扩展所需占位符。
- **事件安全**：所有 `emit` payload 包含一次性 token，前端校验再处理，避免任意网页注入。

## 8. 风险与缓解
| 风险 | 影响 | 缓解 |
| --- | --- | --- |
| WebGL 渲染在低端集显性能抖动 | 动画不丝滑 | 支持 Canvas fallback，低功耗下自动降分辨率 |
| 系统通知权限被禁用 | 用户只看到气泡 | 前端检测失败后显示持久提醒卡片，并提示开启权限 |
| 多屏 DPI 异常导致吸附偏移 | 拖拽体验差 | 在 PlatformBridge 中缓存 `work_area` 并记录最近一次命中用于修正 |
| ReminderEngine 进程退出后任务丢失 | 提醒不可靠 | 所有状态实时写入 SQLite，启动时执行补偿扫描 |

## 9. 验证计划
- **单元测试**：Rust 针对 ReminderEngine 调度、重复规则、补偿逻辑；PlatformBridge Monitor 映射函数。前端测试 MotionController 状态切换守卫。
- **集成测试**：模拟拖拽跨屏、锁屏/解锁、高 DPI 切换、提醒触发与 snooze；自动化脚本通过 Tauri `specta` 或 Playwright 驱动。
- **性能测试**：Chrome DevTools + `tauri::api::process` 统计 CPU/GPU；低功耗模式应至少降低 40% 重绘次数。

## 10. 技术栈与环境依赖
- **前端**：Node.js 20 LTS（本机 v20.19.0）、pnpm 10.x（可换 npm/yarn）、Vite、Vue 3（Composition API）、Pinia、Tailwind CSS、PixiJS/WebGL。
- **后端内核**：Rust stable（本机 rustc 1.94.1）、Tokio、rusqlite、serde、Tauri v2（`cargo-tauri` 2.10.1）。
- **工具链**：Tauri CLI（`cargo install tauri-cli --version ^2`）、SQLite 3.43+（本机 3.43.2）、Git 2.39+、Chrome DevTools、Playwright/tauri-specta 用于测试。
- **macOS 依赖**：Xcode Command Line Tools（`xcode-select -p` 可验证）、Apple Developer 账号用于签名与 Notarization。
- **Windows 依赖（用于跨平台打包/测试）**：Visual Studio Build Tools（含 C++ workload）、WebView2 Runtime、可选 Inno Setup/NSIS。
- **可选辅助**：tokio-console 监控异步任务、TablePlus/DB Browser 查看 `reminders.db`、signtool 或第三方证书做 Windows 签名。
- **初始化步骤**：
  1. 安装 Node.js 20+ 与 pnpm（或 npm/yarn）。
  2. 安装 Rust 工具链：`curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh`，执行 `. $HOME/.cargo/env` 并 `rustup default stable`。
  3. 安装 Tauri CLI：`cargo install tauri-cli --version ^2`。
  4. 确认 SQLite、Git、Xcode CLT/VS Build Tools 已就绪。
  5. （若需 Windows 打包）在 Windows 机器重复上述安装，并配置 WebView2 + 签名工具。
