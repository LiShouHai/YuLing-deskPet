# 隅灵提醒窗口重构设计（2026-04-02）

## 1. 背景
- 用户希望托盘点击“提醒列表”时弹出独立窗口，宠物窗口只展示狗狗且保持静止。
- 现实现方式是在同一 WebView 内绘制提醒面板，导致宠物被挤压、窗口尺寸不够。
- 需要拆分多窗口架构，同时让提醒窗口拥有滚动列表、固定尺寸、后续可扩展能力。

## 2. 目标
1. 新增 `reminder` 子窗口（独立 WebView），默认隐藏，托盘事件调用 `show + focus`。
2. 提醒窗口包含：提醒表单（标题/备注/时间）、提醒列表、操作按钮（完成/延后/删除）。
3. 窗口固定 420×540，内容区域可滚动，但不允许用户拖拽改变尺寸。
4. 原主窗口常态保持 168×168 极简宠物，动画浮动效果取消；当主窗口内出现提醒气泡时，可临时扩展以容纳提示文案，提醒结束后回缩。
5. 维护现有提醒事件、Pinia store、Rust CRUD，不破坏当前数据流。

## 3. 架构与模块
### 3.1 Rust
- `tauri::Builder`: 在 `setup` 中调用 `create_window`：
  - label: `"reminder"`
  - size: `LogicalSize::new(420.0, 540.0)`
  - decorations: `false`, transparent: `true`, always_on_top: `false`, resizable: `false`。
  - 初始 `visible: false`，`center()` 一次。
- 托盘事件 `TRAY_ID_REMINDERS`: `if let Some(win) = app.get_webview_window("reminder") { win.show()?; win.set_focus()?; }`
- 关闭行为：监听 `CloseRequested` 事件改为 `win.hide()`，防止真正销毁；或在 Vue 侧 `invoke("platform_hide_reminder")`。

### 3.2 前端入口
- `src/reminder/main.ts`：创建 Vue 应用并挂载 `ReminderApp.vue`。
- `vite.config.js`: 使用 `build.rollupOptions.input` 指定 `index.html`（宠物）与 `reminder.html`（提醒）。开发模式下 `pnpm dev` 需自定义脚本 `concurrently "vite --config ... --host" "vite --config ... --host --base /reminder"`，或使用单个 Vite，依靠多 HTML entry。
- `reminder.html`: 与 `index.html` 类似，只引用 `src/reminder/main.ts`。

### 3.3 UI 结构
- 顶部：标题 + 副标题 + 发光条（沿用现有样式）。
- 中部：表单（标题、备注、时间、保存按钮、错误提示）。
- 底部：提醒列表，`max-height: 260px; overflow-y: auto;`，列表项复用现有样式。
- 页脚保留操作按钮（完成/延后/删除）。
- 关闭按钮（右上角）触发 `window.appWindow.hide()`。

### 3.4 数据与事件
- 同步复用 `useReminderStore`，将 store 注册在 `reminder/main.ts`。
- 监听 `reminder:fired/updated` 事件保持一致；若 reminder 窗口关闭，事件仍会在后台处理。
- 与宠物窗口共享 `reminderStore`：Tauri 事件+invoke，本质仍是多窗口同一进程，不需额外通信。

### 3.5 宠物窗口调整
- `App.vue`：移除 `@keyframes float` 使用；`pet-body` 不再添加 `animation`，保持静止。
- 删除提醒面板模板相关代码，只保留宠物主体、提醒气泡。

## 4. 交互流程
1. App 启动 → 创建隐藏的 `reminder` window。宠物窗口正常显示。
2. 用户点击托盘“提醒列表” → Rust 端 `show + focus` 提醒窗口。
3. 提醒窗口内的 CRUD 调用 `reminderStore`（通过 Tauri invoke），数据库行为不变。
4. 用户点击关闭按钮或按 ESC → 调用 `@tauri-apps/api/window`. `hide()` 该窗口。宠物窗口不变。

## 5. 测试与验收
- `pnpm tauri dev`：开启两个窗口，无报错；托盘点击一次即可弹出提醒窗；关闭后再次点击仍能弹出。
- 提醒列表超过 6 条时可滚动；各操作能实时更新数据库，并在宠物气泡中体现。
- 宠物窗口无浮动动画、保持静止；提醒窗口关闭后宠物窗口仍在原位置。
- 构建：`pnpm build`、`cargo check` 均通过。
