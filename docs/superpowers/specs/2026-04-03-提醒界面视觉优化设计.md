# 提醒界面视觉优化设计（2026-04-03）

## 背景
- 主窗口提醒气泡过宽、信息过满，提醒触发时容易压住宠物主体。
- 独立提醒窗口采用纵向堆叠，列表区被统计、最近触发、表单等区域挤压，首屏层级不清晰。
- 用户希望提醒界面更有特点，并补充有意义的动效，但不接受喧闹或泛用“科幻面板风”。

## 本轮方向
- 风格：`灵动宠物舱风`
- 主窗口：`轻提醒`
- 提醒窗口：`列表优先`
- 已完成：`默认折叠`

## 主窗口规则
1. 提醒气泡只承载 `标题 + 一行短备注`。
2. 气泡固定为短签，锚定在宠物上方偏右位置，不再居中压住主体。
3. 气泡超长内容直接截断，不通过扩大主窗口去容纳完整文案。
4. 主窗口提醒动效只保留信号浮入和轨道脉冲，不做连续闪烁。

## 提醒窗口规则
1. 待处理提醒作为首屏主区域，并成为唯一滚动区。
2. 统计信息收缩为顶部胶囊，不再使用高占比卡片。
3. 新建提醒降级为可折叠次级区域，由顶部按钮显式展开。
4. 已完成提醒进入单独折叠分组，默认收起，避免与待处理项混排。
5. 最近触发仅保留为轻量信号条，用于辅助感知，不占据主视觉。

## 动效规则
1. 主窗口提醒气泡：220-320ms 浮入/淡出，配合单次环形脉冲。
2. 列表项：使用轻量级联入场，移动与删除优先走 `transform + opacity`。
3. 折叠区：展开/收起使用高度与透明度过渡，不做弹跳。
4. 所有动效必须支持 `prefers-reduced-motion` 回退。

  - 优化提醒脉冲的视觉风格与进出场节奏。

  - 为列表卡片、折叠面板和操作按钮加入克制动效。
