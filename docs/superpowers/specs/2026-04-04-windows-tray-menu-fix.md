# Windows 托盘菜单点击修复（2026-04-04）

## 背景
- 现象：Windows 上点击托盘图标后，菜单能弹出但菜单项点不开；macOS 上同一套逻辑可以使用。
- 影响：`显示隅灵`、`提醒列表`、`切换开机自启`、`退出` 这几个托盘操作在 Windows 上不可用。

## 根因
- 托盘当前同时启用了 `show_menu_on_left_click(true)` 和 `on_tray_icon_event`。
- `on_tray_icon_event` 对任意 `Click` 都会执行主窗口 `show + focus`。
- 在 Windows 上，左键单击托盘图标时会先弹出菜单，再被主窗口抢走焦点，导致用户无法继续点击菜单项。
- macOS 对这组事件的表现更宽松，所以问题没有明显暴露，但逻辑本身依然存在冲突。

## 修复策略
- 托盘单击只负责弹出菜单，不再直接拉起主窗口。
- 保留左键双击图标显示主窗口的快捷行为，避免完全移除托盘图标的直接交互能力。
- 将“是否应该由托盘图标事件拉起主窗口”抽成纯逻辑函数，并补单元测试，避免后续回归。

## 验证
- 自动验证：
  - `cargo test --lib`
  - `cargo check --target-dir target-check`
- 手动验证：
  - Windows：左键点击托盘图标，确认菜单可稳定停留且四个菜单项都可触发。
  - Windows：左键双击托盘图标，确认主窗口可以显示并获得焦点。
  - macOS：确认托盘菜单与 `提醒列表` 打开行为未回退。
